Store base64 images using DRF

Jun 25, 2017 • Thomas Lech  

Hey, in this post we’ll learn how to handle images requested by the users of our app.
These images will be encoded in base64 format.

What is base64 format?

Base64 is a way to represent binary data(stream of 1s and 0s) in an ASCII string format.
It was introduced in order to avoid data corruption when transmitted in binary format across a network.

Examples of binary data corruption:

  • Some protocols may interpret your binary data as control characters (like a modem).
  • The underlying protocol might think that you’ve entered a special character combination (like how FTP translates line endings).

So, after base64 image was successfully transmitted we can decode it back to binary data.
There are two reasons why we are doing that:

  • We want to run image analysis scripts using scikit-image and opencv both of which naturally only support binary input.
  • In order to save image in the database we can only provide DRF with an image encoded in binary format.

After we save the image in the database and our recognizer has already figured out(predicted) what is on the image, we can return prediction back to user.

We are going to do all of this on a server-side using Django and Django REST Framework.

What is Django?

Django is a free and open source web application framework, written in Python. The role of Django boils down to communicating with your app’s database and serving this data to client along with static files. Also, it has a set of handy components that let you for example handle media uploads, form validation or user authentication.

Django project basic scheme:

  • Define Django models and fields.
    Generally, each model maps to a single database table, e.g. users table.
    Fields are table columns, e.g. username, password.
  • Specify URL address and point it to particular view.
  • Build a view
    View defines a behavior after someone entered specific address.

What is Django REST Framework?

Django REST Framework(DRF) is a third-party app for Django. DRF gives you a lot of convenience in inspecting and managing you database(via “Browsable” API). It also does very well in serializing/deserializing data.
